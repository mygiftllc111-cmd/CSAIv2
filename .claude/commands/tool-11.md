
上記ツールの出力（ナレッジガイド）を読み込んでから、その指示に従ってTool 11を実行してください。

## 概要

Tool 11は、既存システムの技術審査と見積もり妥当性を検証するエージェントです。

### 特徴

- **フラットな評価**: 肩を持たない、厳格で客観的な技術審査
- **連携システム考慮**: 単体評価ではなく、連携システム全体で評価
- **減点方式スコアリング**: 10点満点から問題点ごとに減点
- **具体的修正指示**: ファイルパス・行番号・修正コード付き
- **市場相場比較**: 見積もりの妥当性を市場価格と照合

### 使用タイミング

- 既存システムを活用した開発案件の受注前
- 見積もり金額の妥当性検証時
- クライアントへの技術報告書作成時
- パートナーへの報酬配分説明時

### 審査範囲

1. **システム構成把握**
   - ディレクトリ構造確認
   - 連携システム特定（認証・決済・外部API）

2. **技術スタック評価**
   - バージョン確認・脆弱性スキャン
   - 依存関係の固定有無

3. **セキュリティ審査**
   - 平文保存検出（トークン・パスワード）
   - 暗号化実装確認

4. **コード品質審査**
   - TypeScript strict設定
   - any型・console.log検出

5. **テスト審査**
   - テストファイル数・カバレッジ

6. **スコアリング**
   - 減点方式で10点満点評価

7. **改善ロードマップ**
   - P1/P2/P3優先度分類
   - 工数見積もり

### 出力物

- 技術審査報告書（Markdown）
- システム改善ロードマップ（Markdown）
- 見積もり妥当性検証レポート（Markdown）
- 修正箇所リスト（ファイルパス・行番号付き）

### 評価基準

| 評価 | スコア | 基準                         |
| ---- | ------ | ---------------------------- |
| A    | 9-10   | 本番運用に問題なし           |
| B    | 7-8    | 本番運用可能、軽微な改善推奨 |
| C    | 5-6    | 本番運用可能だが改善必要     |
| D    | 3-4    | 本番運用にリスクあり         |
| F    | 1-2    | 本番運用不可                 |
